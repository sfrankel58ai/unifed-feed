1
Create a free JSONBin store
JSONBin gives you a free private URL to store your feed data. Power Automate will write to it; this app will read from it.

1. Go to jsonbin.io and create a free account
2. Click New Bin and paste this as the starting content:
{ "items": [] }
3. Click Create Bin
4. Copy the Bin ID (looks like 64abc123...) and your API Key from the Access Keys section
JSONBIN BIN ID
JSONBIN API KEY (X-MASTER-KEY)
2
Create a Power Automate flow for each account
Repeat these steps for each Outlook account (dispatchbio, adagene, slambio, outlook.com). Power Automate already has permission to access all of them.

1. Go to make.powerautomate.com
2. Click + Create → Scheduled cloud flow
3. Set it to repeat every 15 minutes
4. Add step: Office 365 Outlook → Get emails (V3)
   • Sign in with that account
   • Folder: Inbox, Top: 20, Include Attachments: No
5. Add step: Office 365 Outlook → Get calendar view of events (V3)
   • Start: addDays(utcNow(), -7)
   • End: addDays(utcNow(), 14)
6. Add step: HTTP (to write to JSONBin)
   • Method: PUT
   • URI: (shown below after you save your Bin ID)
   • Headers and body: (shown below)
HTTP ACTION — URI
HTTP ACTION — HEADERS
HTTP ACTION — BODY (paste into Expression editor)
Note: In the HTTP body, replace YOUR_ACCOUNT_EMAIL with the actual email address for that flow's account. Create one flow per account.
3
Run your flows and refresh
Once your flows are created:

1. Click Run on each flow to trigger it immediately
2. Come back here and click ↻ Refresh in the top bar
3. Your emails and calendar events from all accounts will appear in the feed

After the initial run, flows sync automatically every 15 minutes.
